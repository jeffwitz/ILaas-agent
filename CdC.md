# Requirements and Implementation Plan - ILaaS Agent

## 1. Goal

Create an installable GitHub repository that lets users run ILaaS models through local code agents:

- Codex CLI
- Claude Code
- OpenCode

The target user flow is:

```bash
git clone https://github.com/jeffwitz/ILaas-agent.git
cd ILaas-agent
ILAAS_API_KEY="your_ilaas_key" python3 install.py
Ilaas-doctor
Ilaas-opencode run --model qwen-3.6-35b-instruct "Reply exactly: OK"
```

The repository must prioritize a working landing-page experience before explaining internals. Users should be able to install, validate, list models, and run an agent quickly.

## 2. Design Principles

- No real API key is committed to Git.
- The ILaaS key is written only to the local LiteLLM config outside the repository.
- Python owns the portable logic.
- Shell, PowerShell, and CMD wrappers stay thin.
- Dependencies for external agent CLIs are detected by default and installed only when explicitly requested.
- `doctor` does not consume model tokens.
- `smoke` is allowed to consume a small number of tokens and is opt-in.
- Linux is validated first; macOS and Windows native are planned but not yet validated.
- WSL2 is the recommended Windows path for now.

## 3. Architecture

The common backend is ILaaS through LiteLLM Chat Completions:

```text
ILaaS
  <- LiteLLM /v1/chat/completions
    <- Codex Responses proxy for Codex CLI
    <- Claude Messages proxy for Claude Code
    <- OpenCode OpenAI-compatible provider directly
```

Each agent needs a different local interface:

| Agent | Expected interface | Local handling |
| --- | --- | --- |
| Codex CLI | OpenAI Responses API | `codex_ilaas_responses_proxy.py` on port 4001 |
| Claude Code | Anthropic Messages API | `claude_ilaas_messages_proxy.py` on port 4002 |
| OpenCode | OpenAI-compatible provider | Direct LiteLLM provider injected with `OPENCODE_CONFIG_CONTENT` |

## 6. Models

The model list is refreshed from ILaaS `/v1/models`:

```bash
python3 -m ilaas_agents.cli refresh-models
```

Generated aliases:

```text
ilaas-default -> mistral-medium-latest
mistral-ilaas -> mistral-medium-latest
```

Recommended for code-agent use:

```text
qwen-3.6-35b-instruct
mistral-medium-latest
gemma-4-31b
```

Currently not recommended for code-agent tool use:

```text
llama-3.1-8b
llama-3.3-70b
```

## 7. Installer Requirements

The installer must:

1. detect or create the LiteLLM Python virtual environment;
2. install `litellm[proxy]` unless skipped;
3. read the ILaaS API key from `ILAAS_API_KEY` or interactive input;
4. generate the LiteLLM config;
5. refresh the ILaaS model list;
6. generate the Codex config;
7. generate the shared model catalog;
8. install runtime wrappers;
9. optionally check external agent dependencies;
10. optionally install missing external agent CLIs with npm.

Implemented installer options:

```bash
python3 install.py --non-interactive
python3 install.py --skip-litellm-install
python3 install.py --prefix ~/.local
python3 install.py --force
python3 install.py --codex-sandbox-mode workspace-write
python3 install.py --check-agent-deps
python3 install.py --check-agent-deps --install-agent-deps
python3 install.py --check-agent-deps --install-agent-deps --install-agent opencode
```

`--force` creates timestamped backups of generated files before overwriting them.

## 8. Dependency Handling

External CLIs are detected by default and installed only on explicit request.

Detection:

```bash
python3 -m ilaas_agents.cli deps status
```

Opt-in install:

```bash
python3 -m ilaas_agents.cli deps install all
python3 -m ilaas_agents.cli deps install codex
python3 -m ilaas_agents.cli deps install claude
python3 -m ilaas_agents.cli deps install opencode
```

Packages used:

```text
codex    -> npm install -g @openai/codex
claude   -> npm install -g @anthropic-ai/claude-code
opencode -> npm install -g opencode-ai
```

The npm package named `codex` must not be used for OpenAI Codex CLI.

## 9. Runtime Commands

```bash
Ilaas-doctor
Ilaas-servers status
Ilaas-servers start
Ilaas-servers stop
Ilaas-servers logs
Ilaas-codex exec --skip-git-repo-check "Reply exactly: OK"
Ilaas-claude -p --model qwen-3.6-35b-instruct "Reply exactly: OK"
Ilaas-opencode run --model qwen-3.6-35b-instruct "Reply exactly: OK"
```

Token-consuming smoke tests:

```bash
python3 -m ilaas_agents.cli smoke --agent opencode --model qwen-3.6-35b-instruct
python3 -m ilaas_agents.cli smoke --agent all --model qwen-3.6-35b-instruct
```

## 10. Validation Requirements

Current checks:

```bash
python3 -m py_compile install.py ilaas_agents/*.py proxies/*.py scripts/clone_isolated_check.py
python3 -m unittest discover -s tests
bash -n Ilaas-codex Ilaas-claude Ilaas-opencode Ilaas-doctor Ilaas-servers install.sh
python3 -m sphinx -b html -W --keep-going docs docs/_build/html
python3 scripts/clone_isolated_check.py
```

Validated locally:

- clone from GitHub into `/tmp`;
- unit tests from the clone;
- isolated install into `/tmp` using path overrides;
- generated wrappers from the isolated install;
- `Ilaas-doctor` from isolated paths;
- `Ilaas-opencode` model call with Qwen from isolated paths.
