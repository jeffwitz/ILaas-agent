# Windows

Recommended path today: WSL2.

The repository is designed for multi-OS use, but only Linux has been validated end to end. Windows native wrappers are generated by `install.py`, but they are not yet validated with Codex, Claude Code, and OpenCode.

## Wine Smoke Test

Linux developers can run a limited Wine smoke test for generated `.cmd` wrappers:

```bash
python scripts/wine_windows_wrapper_check.py
```

This verifies that generated `.cmd` files execute under Wine `cmd.exe`, set `PYTHONPATH`, and call:

```text
python -m ilaas_agents.cli <agent>
```

This is not a replacement for native Windows validation. It does not validate a real Windows Python installation, PowerShell wrappers, Codex, Claude Code, OpenCode, LiteLLM, or Windows filesystem behavior.

Windows native planned paths:

```text
LiteLLM config: %APPDATA%\litellm\ilaas-mistral.yaml
Codex home: %USERPROFILE%\.codex-ilaas
Wrappers: %LOCALAPPDATA%\Programs\IlaasCodeAgents\bin
Logs: %LOCALAPPDATA%\ilaas-code-agents\logs
```

Generated wrapper types:

```text
Ilaas-codex.cmd / Ilaas-codex.ps1
Ilaas-claude.cmd / Ilaas-claude.ps1
Ilaas-opencode.cmd / Ilaas-opencode.ps1
Ilaas-doctor.cmd / Ilaas-doctor.ps1
Ilaas-servers.cmd / Ilaas-servers.ps1
```

Before marking Windows native as supported, validate:

```powershell
python install.py --non-interactive --skip-litellm-install
Ilaas-doctor
Ilaas-opencode run --model qwen-3.6-35b-instruct "Reply exactly: OK"
```
